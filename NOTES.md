## ValidationPipe

**File:** `src/main.ts`

```ts
app.useGlobalPipes(new ValidationPipe({ whitelist: true, forbidNonWhitelisted: true }));
```

Registers `class-validator` decorators globally — every `@Body()` DTO is validated **before** the controller runs.

### Why necessary
NestJS does not auto-validate DTOs. Without `ValidationPipe`, the `@IsEmail()`, `@IsNotEmpty()`, etc. decorators in DTO classes are ignored entirely — invalid data reaches your service.

### Options used
| Option | Effect |
|---|---|
| `whitelist: true` | Strips any properties that don't have a decorator. Blocks injection attacks via extra fields. |
| `forbidNonWhitelisted: true` | Throws `400` if the client sends unexpected properties (stricter than just stripping). |

### How it connects to Mongoose
`ValidationPipe` runs on the **DTO** (the incoming HTTP request body). Mongoose runs validations on the **schema** (`@Prop()` decorators) when saving to MongoDB. Two layers of validation:

```
HTTP Body → DTO (class-validator) → Controller → Service → Mongoose Schema (@Prop) → MongoDB
```

## Mongoose Schema vs TypeORM Entity

| Concept | Mongoose (this project) | TypeORM (not used) |
|---|---|---|
| Decorator | `@Schema()`, `@Prop()` | `@Entity()`, `@Column()` |
| Module registration | `MongooseModule.forFeature([{ name: 'Name', schema: NameSchema }])` | `TypeOrmModule.forFeature([Entity])` |
| Service injection | `@InjectModel('Name') model: Model<DocType>` | `@InjectRepository(Entity) repo: Repository<Entity>` |
| CRUD methods | `model.find()`, `model.findById()`, `model.save()` | `repo.find()`, `repo.findOne()`, `repo.save()` |
| Auto-generated ID | `_id` (ObjectId — 24 hex chars) | Auto-increment integer |

## Key Concepts Learned

### DTOs (Data Transfer Objects)
Define the shape of incoming request data. `class-validator` decorators validate fields. `PartialType(CreateUserDto)` generates an update variant where all fields are optional.

### Guards (learned conceptually, not implemented yet)
Layer of route protection. `JwtAuthGuard` validates the token. `RolesGuard` checks user role. Both are `@UseGuards()` decorators on controller methods.

```
Request → JwtAuthGuard (token valid?) → RolesGuard (role correct?) → Handler
```

## Common Errors Encountered

| Error | Cause | Fix |
|---|---|---|
| `ValidationPipe` not registered → DTO validators ignored | Missing `app.useGlobalPipes(...)` in `main.ts` | Add it — DTOs are useless without it |
| TypeORM entity pattern (`@Entity`, `@Column`) generated by Nest CLI | Nest CLI scaffolds TypeORM by default | Delete entity file, replace with Mongoose `@Schema()/@Prop()` |
| `+id` number coercion in controller params | Nest CLI generates `findOne(+id)` for SQL numeric PKs | Remove `+`, pass string directly for MongoDB ObjectId |
| `@nestjs/mongoose/dist/mongoose.module` deep import | Auto-import picks wrong path | Import from `@nestjs/mongoose` |
